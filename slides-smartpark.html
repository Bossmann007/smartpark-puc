O Problema

Quem nunca ficou rodando no estacionamento da PUC?

  • Os totens da entrada só mostram "lotado" ou "disponível" — não onde tem vaga
  • Motorista entra, percorre o corredor todo, não tem nada — tem que voltar
  • Motor ligado o tempo todo = CO₂ desnecessário
  • Até 30% do tráfego urbano é causado por busca de estacionamento
2 / 9
A Solução

Um sensor por vaga. Um LED. Resolvido.

  • Sensor ultrassônico em cada vaga detecta se tem carro
  • LED verde = livre · LED vermelho = ocupada
  • Buzzer avisa quando o estacionamento está lotado
  • Dados em tempo real via MQTT pro dashboard Processing
HC-SR04 (vaga 1 e 2) ↓ GPIO 5/18 · 19/21 ESP32 DevKit V1 ↓ GPIO 2/4/22/23 4× LED (verde + vermelho) ↓ GPIO 25 Buzzer — estac. lotado ↓ WiFi / MQTT Dashboard Processing

Arquitetura CPS

As quatro camadas do sistema ciberfísico

Camada Componente O que faz
Sensing HC-SR04 Mede distância até o carro (mundo físico → digital)
Computation ESP32 + MicroPython Decide se a vaga está ocupada (dist < 20 cm)
Communication WiFi + MQTT Envia o estado das vagas pro dashboard em tempo real
Actuation LED + Buzzer + Dashboard Sinaliza visualmente, sonoramente e na tela (digital → físico)

MicroPython no ESP32

Firmware — vetores, matrizes e laços

# vetor: pinos dos sensores
trig = [Pin(5,OUT), Pin(19,OUT)]
echo = [Pin(18,IN),  Pin(21,IN)]

# matriz: leds[vaga][0]=verde [1]=verm.
leds = [[Pin(2,OUT), Pin(4,OUT)],
        [Pin(22,OUT),Pin(23,OUT)]]

for i in range(2):
    d = medir(trig[i], echo[i])
    if d < LIMIAR:
        leds[i][0].off()
        leds[i][1].on()   # vermelho
    else:
        leds[i][0].on()    # verde
        leds[i][1].off()

if ocup[0] and ocup[1]:
    buzzer.duty(512)  # lotado
  • Vetor (Tópico 06): trig[] e echo[] — um índice por vaga
  • Matriz (Tópico 07): leds[vaga][cor] — acessa verde ou vermelho por posição
  • for (Tópico 05): itera as 2 vagas sem repetir código
  • if/elif/else (Tópico 02/03): decide LED e buzzer
  • Se cair o WiFi: LEDs e buzzer continuam normalmente

Impacto

Por que isso importa

  • Menos tempo rodando à toa = menos CO₂
  • Custo por vaga monitorada: ~R$ 35
  • Fácil de escalar: mais ESP32 na mesma rede MQTT
  • Mesma arquitetura funciona pra outros recursos: salas, bikes, vagas PcD
ANTES: Motorista circula sem informação → CO₂ + tempo perdido DEPOIS: LED verde = vai direto LED vermelho = não entra Buzzer = avisa admin Dashboard = visão geral
